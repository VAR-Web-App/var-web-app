## 8. Channel Ops Platform — Data Access & Integration

The tool sits on top of four data layers. Layer 1 (Cisco) is the moat. Layer 3 (federal public) is buildable today with no dependencies. Layers 2 + 4 require Avanchor's grants.

### Layer 1 — Cisco APIs (the moat)

#### 1.1 CCW (Cisco Commerce Workspace)

| Endpoint | Purpose | Used for |
|---|---|---|
| Estimate API | Build estimates from BOMs | Quote engine |
| Quote API | Convert estimate → quote | Quote workflow |
| Pricing API | Partner-specific pricing | Quote engine + PVI deal scoring |
| Order API | Submit / track orders | Order pipeline |
| Configuration API | Validate Cisco product configs | Deal scoring |

**Access requirements:**
- Cisco partner account in good standing (Avanchor: ✅ Premier)
- developer.cisco.com (DevNet) developer profile linked to Avanchor's partner account
- OAuth 2.0 client credentials generated from DevNet
- Per-API entitlement (some auto-approve, others need partner support ticket)
- Sandbox available; production requires additional approval

**Premier-tier specifics:**
- More CCW Pricing API depth than Select; less than Gold
- Sufficient for partner-specific pricing across the full product catalog
- Some advanced pricing endpoints (deal-specific overrides, special bid pricing) may be Gold-gated — confirm with Bobby

#### 1.2 CCW-R (Renewals)

**What it gives:** installed base per customer, service contracts + expirations, renewal opportunities, Smart Service / Solution Support entitlements, co-term opportunities.

**Critical because:** Renewals are 80%+ margin. Distributors poach passive renewals. CCW-R API beats them to the customer.

**Access:** same Cisco partner account + DevNet profile + separate API entitlement. Renewals Manager role on partner account often required.

**Avanchor-specific risk:** if historic service attach was inconsistent, installed base will have gaps (orphaned installs). Mitigate by treating CCW-R data as "best available, subject to attach quality."

#### 1.3 EA Workspace API

**What it gives:** EA status per customer, consumption/usage, renewal opportunities, true-up obligations.

**PVI relevance:** EA attach is one of the highest-impact PVI levers. The PVI Optimizer recommends "add an EA" when deal characteristics fit — needs this data to track outcomes.

**Access:** Partner authorization + customer must have an active EA. Customer-grant authorization sometimes required.

**Status check:** ask Bobby whether Avanchor sells EAs today. If not, EA module is Phase 2+ (build infrastructure now, populate later).

#### 1.4 Smart Account / Smart Licensing API

**What it gives:** Smart Account hierarchy per customer, license entitlement + consumption, lifecycle status, DNA license tracking.

**PVI relevance:** Smart Account adoption is itself a PVI component. Smart Licensing data drives upsell intelligence.

**Access:** Smart Account admin role on customer accounts (Bobby confirms count); customer-grant authorization for sensitive data.

#### 1.5 Cisco Partner Self-Service / Program data (PVI source-of-truth)

**Reality:** Cisco does not publish a comprehensive public API for PVI scoring data. Visible in Partner Self-Service web UI, but programmatic access is limited.

**Strategy: Shadow PVI engine + calibration.**
1. Compute estimated PVI from CCW + CCW-R + manual specialization roster using known component weights
2. Calibrate against actual PVI when Cisco publishes monthly
3. Adjust weights based on observed correlation
4. After 2–3 calibration cycles, accuracy is ±0.1 — directionally sufficient

**Fallback:** Bobby exports PVI data quarterly; we ingest manually. Less elegant but reliable.

#### 1.6 Engineer Certifications

**Reality:** per-individual data, no aggregated partner API.

**Approach:** manual roster maintained by Bobby (export from Cert Tracker, upload to tool). Phase 2 may automate if Cisco enables it.

### Risks / unknowns specific to access

1. **Premier tier limits some pricing endpoints.** Tool still works; deal-scoring precision slightly lower than Gold-tier partners would have.
2. **CCW-R historic data quality.** Depends on Avanchor's past attach discipline. Plan for incomplete installed base initially.
3. **Cisco rate limits.** Fine for single-partner tool; watch for cumulative limits if scaled to many partners.
4. **Distributor API quality varies.** Ingram > TD SYNNEX > Carahsoft (closed). Plan manual fallback for some.
5. **Cisco program changes annually.** PVI components shift. Shadow PVI needs ongoing recalibration. Build flexibility into scoring rules from day 1.
6. **Customer Smart Account authorization slow.** Per-customer grants required for some data; some accounts will lag.
